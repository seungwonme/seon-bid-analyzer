# 입찰 공고 분석기 PRD

## 개요

나라장터 입찰 공고 PDF를 업로드하면 핵심 정보를 카드로 정리해 주고, 제출서류 체크리스트와 자격서류 작성·내보내기까지 지원하는 웹 도구. 로그인 없이 브라우저에서 즉시 사용 가능하며, 데이터는 localStorage에 유지된다.

**대상 사용자**: 선엔지니어링 수주전략팀 (비개발자 포함)  
**기술 스택**: Next.js (App Router), React, TypeScript, Tailwind CSS  
**데이터 저장**: localStorage (서버 DB 없음)  
**AI·외부 API**: 사용 안 함 (PDF.js로 텍스트 추출 → 정규식·키워드 파싱)

---

## 핵심 원칙

- **"원문 확인 필요" 경고**를 추출 결과 최상단에 항상 표시한다. 자동 추출은 보조 수단이며 최종 확인은 사용자가 한다.
- 로그인·회원가입 없음.
- 단순함 우선: 버튼 하나로 시작, 전문 용어 최소화.

---

## 기능 요구사항

### F1. PDF 업로드 및 정보 추출

- 공고 PDF를 드래그 앤 드롭 또는 파일 선택으로 업로드
- PDF.js로 전체 텍스트 추출 후 아래 필드를 정규식·키워드 매칭으로 파싱

| 필드 | 추출 키워드 예시 |
|------|----------------|
| 공고명 | 공고명, 용역명 |
| 발주기관 | 발주기관, 수요기관 |
| 예정금액 | 예정가격, 추정가격 |
| 용역기간 | 용역기간, 계약기간 |
| 입찰 마감일시 | 입찰서 제출마감, 마감일 |
| 참가자격 | 참가자격, 자격요건 |
| 제출서류 목록 | 제출서류, 입찰서류 |

- 추출 실패 필드는 "미확인 — 원문 직접 확인" 표시
- 업로드 직후 황색 배너로 **"AI가 아닌 키워드 추출 방식입니다. 반드시 원문을 확인하세요."** 고정 표시

### F2. ERP 대시보드 뷰

**레이아웃**: 좌측 사이드바 공고 목록 + 우측 메인 패널 고정 분할 (SAP 스타일)

**사이드바 (공고 목록)**
- 행 1개 = 공고 1개: 공고명(줄임) + D-Day 뱃지 + 체크리스트 완료율 진행바
- 마감일 기준 자동 오름차순 정렬
- 선택된 공고 하이라이트
- 공고가 없을 때: 업로드 안내 문구

**메인 패널 (공고 상세)**
- 상단: 공고명 + 발주기관 + D-Day 뱃지
- 탭 3개 (슬라이드 패널 없이 인라인):
  - `개요` — 추출 필드 전체를 키-값 테이블로 표시
  - `체크리스트` — 상태 토글·항목 추가·삭제·진행률 바
  - `서류작성` — 회사 정보·실적·경력 폼 + docx 내보내기
- 공고 미선택 시: "좌측에서 공고를 선택하거나 PDF를 업로드하세요" 안내

**D-Day 뱃지 기준**
- 7일 초과: 초록
- 7일 이하: 빨간색 + 강조
- 마감 당일: "D-Day" 빨간 굵은 글씨
- 마감 지남: "마감" 회색
- 미확인: "미확인" 회색

### F3. 제출서류 체크리스트

- 추출된 제출서류 목록을 체크박스 리스트로 표시
- 각 항목에 "완료 / 준비 중 / 미시작" 상태 선택
- 전체 진행률(%) 프로그레스 바로 시각화
- 항목 수동 추가·삭제 가능 (추출 누락 대비)
- 상태는 localStorage에 자동 저장 (새로고침 유지)

### F4. ~~다중 공고 비교 뷰~~ (v1 제외 → 사이드바 목록으로 대체)

### F5. 자격서류 작성 및 내보내기

- 탭 또는 모달로 접근하는 폼 에디터
- 폼 항목:
  - 유사용역 실적: 용역명, 발주기관, 계약금액, 용역기간, 담당업무 (행 추가 가능)
  - 기술자 경력: 성명, 자격종목, 취득일, 소속, 담당분야 (행 추가 가능)
  - 회사명·사업자등록번호 (공통 헤더, localStorage 자동 저장)
- 내보내기: **Word(.docx)** 형식 (docx 생성 라이브러리 사용)
- 입력 내용은 localStorage에 자동 저장

## 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 로딩 속도 | PDF 업로드 후 카드 표시까지 5초 이내 |
| 반응형 | 데스크톱(1280px) 우선, 태블릿(768px) 지원 |
| 브라우저 | Chrome 최신, Edge 최신 |
| 접근성 | 색만으로 정보 전달하지 않음 (D-Day 색+텍스트 병행) |
| 데이터 보안 | PDF 및 입력 데이터는 브라우저 외부로 전송하지 않음 |

## 화면 구성 (와이어프레임 수준)

SAP 스타일 ERP 대시보드. 좌측 공고 목록 + 우측 상세 패널 고정 레이아웃.

```
┌──────────────────────────────────────────────────────────────┐
│ 헤더: 입찰 공고 분석기 | 선엔지니어링          [+ PDF 업로드] │
│ ⚠ 키워드 추출 방식입니다. 반드시 원문을 확인하세요.           │
├──────────────────┬───────────────────────────────────────────┤
│ 공고 목록        │ 공고명 (선택된 공고)            D-3 🔴    │
│ (마감일순 정렬)  │ 발주기관                                  │
│                  │ ──────────────────────────────────────── │
│ ▶ 청주지사 공사  │ [개요]  [체크리스트]  [서류작성]          │
│   D-3  ████ 43%  │                                          │
│                  │ 탭 콘텐츠 (인라인, 슬라이드 패널 없음)    │
│   인천본부 용역  │                                          │
│   D-15 ██   20%  │ 개요 탭: 추출 필드 키-값 테이블           │
│                  │ 체크리스트 탭: 상태 토글·추가·삭제        │
│   서울지사 공사  │ 서류작성 탭: 실적·경력 폼 + docx 내보내기 │
│   D-30      0%   │                                          │
└──────────────────┴───────────────────────────────────────────┘
```

**레이아웃 원칙**
- 사이드바 고정 너비 (~280px), 나머지는 메인 패널
- 공고 없을 때: 메인 패널에 업로드 안내 표시
- 슬라이드 오버 패널 제거 → 모든 기능이 탭 안에서 인라인 처리
- 비교 뷰 제거 (10개+ 환경에서 사용성 낮음)
